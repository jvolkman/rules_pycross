# Architecture: Build Profiles & Mixins

This document outlines the compositional architecture of `rules_pycross` v2's PEP 517 build pipeline.

### Build Profiles

Build Profiles are public-facing Starlark macros (e.g., `maturin_build`, `meson_build`, `setuptools_build`) that orchestrate PEP 517 builds. They accept standard Bazel attributes like `deps` and `native_deps` and map them to the appropriate internal toolchains. They dynamically instantiate the required Mixins and pass them to the core `pycross_pep517_build` rule.

### Mixins

Mixins are private, reusable, backend-agnostic Starlark rules (e.g., `pycross_cc_mixin`, `pycross_rust_mixin`) that resolve specific Bazel toolchains (C++, Rust) and extract their compiler paths, sysroots, and flags. They package these variables into a JSON configuration file exposed via a `PycrossBuildMixinInfo` provider.

### Builder Scripts

Builder Scripts are Python scripts (e.g., `maturin_builder.py`, `meson_builder.py`) that run in the Bazel sandbox to parse JSON configurations generated by Mixins and translate them into backend-specific environments. A builder script is responsible for assembling a build virtual environment and performing the build, as well as integrating any user-provided hooks.
